## 1. 两个正交维度（必须先理解）

主题由两个**互相独立**的维度组成，分别存取、可自由组合：

| 维度 | 取值 | 写到哪 | 持久化 |
|---|---|---|---|
| **Mode（明暗）** | `light` / `dark` / `system` | `<html class="dark">`（由 next-themes 控制） | `localStorage`（next-themes 自管）+ 后端 |
| **Preset（配色）** | 59 个 shadcn 主题 key（`blue` / `claude` / `supabase` …）或 `null`（默认黑白） | `<html data-theme="blue">` | `localStorage("color-preset")` + 后端 |

两者能组合，是因为 `themes.generated.css` 给每个主题**同时**生成 light 和 dark 两条规则（见 §4.2）：

```css
:root[data-theme="blue"]      { --primary: ...; ... }   /* blue + light */
:root.dark[data-theme="blue"] { --primary: ...; ... }   /* blue + dark  */
```

> 记住这条：**Mode 切 `class`，Preset 切 `data-theme`**，CSS 选择器优先级让 `data-theme` 的值覆盖基础 token。复刻时这两个属性的落点一个都不能错。

## 2. 数据流全景图

```
┌─ 用户在 ThemeDrawer 操作 ────────────────────────────────────┐
│  useThemeSync()  ← 唯一写入口，暴露 setMode / setPreset / reset │
│    ├─ setMode(mode):                                          │
│    │     next-themes setTheme(mode)  → <html class>           │
│    │     orpc me.preferences.update.mutate({ themeMode })     │
│    ├─ setPreset(preset):                                      │
│    │     document.documentElement.dataset.theme = preset      │
│    │     localStorage["color-preset"] = preset                │
│    │     orpc ...update.mutate({ themePreset })               │
│    └─ reset(): 清掉 data-theme + localStorage + mode=system   │
└──────────────────────────────────────────────────────────────┘

┌─ 页面加载时恢复（两条路径，缺一不可）────────────────────────┐
│  A. <head> 内联脚本（__root.tsx，防 FOUC）                     │
│       try{ p=localStorage["color-preset"]; if(p)              │
│            html.setAttribute("data-theme", p) }catch{}        │
│       ↑ 阻塞式、首屏渲染前执行 → 不闪白/不闪默认主题            │
│  B. ThemeSyncBoundary（仅登录用户）                            │
│       useQuery me.preferences.get → 一次性把服务端偏好应用到 DOM│
│       ↑ 跨设备同步；用 appliedRef 保证"只应用一次"             │
└──────────────────────────────────────────────────────────────┘

后端：user.preferences (jsonb) ← me.preferences service ← user-repository
```

**两条恢复路径的分工**：内联脚本负责**本地 localStorage 的即时恢复**（防闪烁，匿名用户也生效）；`ThemeSyncBoundary` 负责**登录用户的跨设备恢复**（从数据库拉）。

## 4. CSS 层 — 值的来源

### 4.1 `globals.css`（基础 token，手写）

结构（顶部到底部）：

```css
@import "tailwindcss";
@import "tw-animate-css";
@import "shadcn/tailwind.css";
@import "@fontsource-variable/inter";
@import "@fontsource-variable/noto-sans-sc";
@import "./themes.generated.css";          /* ← 主题覆盖层，必须在基础 token 之后 import */
@source "../../../apps/**/*.{ts,tsx}";      /* Tailwind v4 内容扫描 */
@source "../**/*.{ts,tsx}";

@custom-variant dark (&:is(.dark *));        /* dark: 变体绑定 .dark class */

:root {                                      /* light 基础 token */
  --background, --foreground, --primary, --secondary, --muted, --accent,
  --destructive, --border, --input, --ring, --chart-1..5,
  --radius: 0.5rem,
  --sidebar-*,                               /* 侧边栏 8 个 token，见 sidebar.md §7 */
  --up/--down/--flat,                        /* 语义色：涨跌（红涨绿跌，CN 习惯）*/
  --info/--warn/--success,                   /* 语义色：严重度 */
}
.dark { /* 对应一套深色基础 token */ }

@theme inline {                              /* 把 CSS 变量映射成 Tailwind 工具类 */
  --color-primary: var(--primary);
  --color-sidebar: var(--sidebar);
  --radius-sm: calc(var(--radius) - 4px);    /* 圆角档位都从 --radius 派生 */
  --radius-md: calc(var(--radius) - 2px);
  --radius-lg: var(--radius);
  /* ... */
}
@layer base { /* 全局基础样式 */ }
@layer utilities { /* 自定义工具类 */ }
```

要点：

- **`@theme inline` 是关键**：只有在这里 `--color-x: var(--x)` 映射过的 token，才能用 `bg-x`/`text-x` 工具类。新增可主题化颜色必须两步——先在 `:root` 定义 `--x`，再在 `@theme` 映射 `--color-x`。
- **圆角统一**：`--radius` 是单一来源，`--radius-sm/md/lg/...` 全部 `calc()` 派生。改圆角只动 `--radius`。这也是侧边栏文档里说"圆角跟随全局 `--radius`"的出处。
- **import 顺序**：`themes.generated.css` 必须在基础 `:root` 之后被 import（实际是同文件靠后/或被 `@import` 引入后由选择器优先级覆盖）——`[data-theme]` 选择器比裸 `:root` 更具体，所以能覆盖。

### 4.2 `themes.generated.css`（生成，勿手改）

每个主题两条规则：

```css
/* generated by sync-shadcn-themes.ts — do not edit */
:root[data-theme="blue"]      { --background: ...; --primary: ...; /* 全套 token */ }
:root.dark[data-theme="blue"] { --background: ...; --primary: ...; }
```

当 `<html data-theme="blue">` 时，这些规则覆盖 `globals.css` 的基础 token；`<html class="dark" data-theme="blue">` 时命中 `:root.dark[data-theme="blue"]`。没有 `data-theme` 时回落到基础黑白 token。

## 5. 生成器 — `pnpm theme:sync`

### 5.1 `theme-categories.ts`（手工维护）

- `Category` 联合类型（11 个：color/pastel/dark/minimal/creative/warm/tech/brand/luxury/retro/nature）。
- `CATEGORIES: Record<ThemeKey, Category>`：每个主题 key 归类。
- `THEME_KEYS` / `ThemeKey`：所有主题 key 的清单与类型。

### 5.2 `sync-shadcn-themes.ts`（`packages/ui` 的 `theme:sync` 脚本）

流程：

1. **硬校验** `THEME_KEYS.length === 59`，不符直接抛错（加主题必须同步改这个数字）。
2. 分批（每批 6 个）从 `https://shadcn.io/r/{key}.json` 拉取，带重试（最多 3 次，指数退避）。可选 `SHADCN_API_TOKEN` 环境变量加到 query。
3. 校验响应含 `cssVars.light` 和 `cssVars.dark`。
4. 产出两个文件：
   - `themes.generated.css`：每主题 `toCssBlock(":root[data-theme=key]", light)` + `toCssBlock(":root.dark[data-theme=key]", dark)`。
   - `themes.generated.ts`：`THEMES` 数组（每条含 `key` / `name` / `category`（查 `CATEGORIES`）/ `swatches`（只取 `background`/`primary`/`accent`/`foreground` 四色，light+dark 各一组，给 UI 卡片用）），并导出 `ThemeKey = (typeof THEMES)[number]["key"]`。

> 设计意图：**单一生成出口**。CSS（注入 DOM 的实际值）和 TS（UI 用的注册表/类型/swatch）来自同一次抓取，永远一致。手改任一个都会在下次 sync 被覆盖。

## 6. React 层

### 6.1 `theme-provider.tsx`

```tsx
<NextThemesProvider attribute="class" defaultTheme="system" disableTransitionOnChange enableSystem>
```

- `attribute="class"` → mode 写到 `<html class>`（与 `@custom-variant dark (&:is(.dark *))` 对齐）。
- `disableTransitionOnChange` → 切换瞬间禁用 CSS transition，避免颜色渐变拖影。

### 6.2 `use-theme-sync.ts`（核心 hook）

- 读 next-themes 的 `setTheme`，读 `useThemeOrpc()` 拿 app 注入的 orpc client。
- `currentPreset` state 初值从 `localStorage["color-preset"]` 读。
- **跨标签页同步**：`useEffect` 监听 `storage` 事件，别的标签改了 preset，本标签同步更新 state + DOM。
- `update` mutation：`orpc.me.preferences.update`，`onError` 时 `toast.error("...applied locally")`——**写后端失败不阻断本地生效**（乐观、本地优先）。
- 三个动作：
  - `setMode(mode)`：`setTheme(mode)` + `mutate({ themeMode })`。
  - `setPreset(preset)`：写 DOM `dataset.theme` + 写 localStorage + setState + `mutate({ themePreset })`。
  - `reset()`：mode 回 `system` + 清 DOM data-theme + 清 localStorage + `mutate({ themePreset: undefined, themeMode: "system" })`。

### 6.3 `theme-sync-boundary.tsx`（登录用户恢复）

- `useQuery me.preferences.get`，`enabled` 绑定"是否登录"，`staleTime: Infinity`。
- **只应用一次**：用 `appliedRef` 守卫；用 `setThemeRef` 持有最新 `setTheme`。
  > 注释解释得很清楚：next-themes 每次主题变更都会重建 `setTheme`（useCallback deps 含当前主题），若把 `setTheme` 放进 effect deps，会在用户每次点击后重跑 sync，把主题**弹回服务端值**。所以只在 `data` 首次到达时应用。
- 应用：`setThemeRef.current(themeMode)` + `dataset.theme = themePreset` + 写 localStorage。

### 6.4 `orpc-context.tsx`（解耦关键）

`packages/ui` **不能**直接依赖某个 app 的 orpc client（会形成循环/反向依赖）。所以定义最小接口 `ThemeOrpcClient { me.preferences.get/update }`，由各 app 在 `__root.tsx` 用 `<ThemeOrpcProvider client={orpc}>` 注入，UI 内用 `useThemeOrpc()` 取。

### 6.5 抽屉 UI

- `theme-drawer.tsx`：`<Sheet side="right" w-[400px]>`，三段：**Mode**（`ThemeModeToggle`）/ **Color**（`ThemeGrid`）/ **Reset 按钮**。从 `useThemeSync()` 取四个值。
- `theme-mode-toggle.tsx`：Light/Dark/System 三个等宽 `h-8` 按钮，当前项 `variant="default"` + `ring-2`。
- `theme-grid.tsx`：按 `CATEGORY_ORDER` 分组渲染，每组标题显示 label + 数量，`grid-cols-2` 卡片。
- `theme-card.tsx`：渲染 4 个色点（按当前 `resolvedTheme` 选 light/dark swatch），选中态 `border-primary ring-2` + 右上角 Check。

## 7. 后端持久化

- **存储**：`user.preferences` 是 `jsonb` 列（schema `packages/db/src/schema/auth.ts`，迁移 `0009`）。Schema-on-read——加字段不需要迁移。
- **形状**（`preferences-types.ts` 的 `UserPreferencesSchema`，`.strict()`）：
  ```ts
  { themePreset?: string; themeMode?: "light"|"dark"|"system"; onboarding?: {...} }
  ```
- **service**（`preferences-service.ts`）：
  - `getPreferences(userId)` → `{ themePreset, themeMode }`（null 兜底）。
  - `updatePreferences(userId, patch)` → 先读现有再 `{ ...existing, ...patch }` **merge** 后整体写回（所以单独 `mutate({ themeMode })` 不会清掉 `themePreset`）。
- **repository**：`getUserPreferences` 读 `row.preferences ?? {}`；`setUserPreferences` 整列 `update`。
- **router**：`me.preferences.get` / `me.preferences.update`，遵循 `.claude/rules/backend.md` 的 services+routers 分层（router 只绑 zod + 调 service）。

## 8. 接入 — 各 app 的 `__root.tsx`

两个 app 完全对称：

```tsx
head: () => ({
  scripts: [{
    // 防 FOUC：首屏渲染前同步把 localStorage 的 preset 写进 data-theme
    children: 'try{var p=localStorage.getItem("color-preset");if(p)document.documentElement.setAttribute("data-theme",p);}catch(e){}',
  }],
}),

function RootDocument() {
  const session = authClient.useSession();
  return (
    <html lang="en" suppressHydrationWarning>            {/* next-themes 改 class，必须 suppress */}
      <head><HeadContent /></head>
      <body>
        <ThemeOrpcProvider client={orpc as never}>       {/* 注入 orpc，解耦 */}
          <ThemeProvider>                                 {/* next-themes */}
            <ThemeSyncBoundary enabled={!!session.data?.user}>  {/* 登录才拉服务端偏好 */}
              {/* ...AnimatedOutlet + Toaster... */}
            </ThemeSyncBoundary>
          </ThemeProvider>
        </ThemeOrpcProvider>
      </body>
    </html>
  );
}
```

要点：

1. **`suppressHydrationWarning`** 必加在 `<html>`：内联脚本/next-themes 会在 hydration 前改 `class`/`data-theme`，否则 React 报 mismatch。
2. 嵌套顺序固定：`ThemeOrpcProvider → ThemeProvider → ThemeSyncBoundary`。
3. **`<head>` 内联防闪烁脚本不可省**：没有它，登录用户首屏会先渲染默认主题，等 `ThemeSyncBoundary` 的 query 回来才跳变。脚本只处理 preset（mode 由 next-themes 自己的内联脚本处理）。
4. `ThemeDrawer`（实际触发器 `ThemeDrawerTrigger`）放在各自 layout 顶栏（admin/web 的 `routes/admin.tsx`/`app.tsx`，见 sidebar.md §8）。

## 9. 复刻 checklist（按序）

1. **依赖**：`packages/ui` 装 `next-themes`（catalog）、`@tanstack/react-query`、`sonner`、`lucide-react`。
2. **CSS 基础层**：`globals.css` 写基础 `:root`/`.dark` token + `@theme inline` 映射 + `@custom-variant dark`，`@import "./themes.generated.css"`。
3. **生成器**：落 `scripts/theme-categories.ts`（key→category）+ `scripts/sync-shadcn-themes.ts`，`package.json` 加 `"theme:sync": "tsx scripts/sync-shadcn-themes.ts"`，跑一次生成 `themes.generated.{css,ts}`。
4. **next-themes**：`theme-provider.tsx`（`attribute="class"` + `disableTransitionOnChange`）。
5. **orpc 解耦**：`orpc-context.tsx`（Provider + hook + 最小接口）。
6. **核心 hook**：`use-theme-sync.ts`（三动作 + storage 同步 + 乐观写后端）。
7. **恢复**：`theme-sync-boundary.tsx`（appliedRef/setThemeRef 防回弹）。
8. **UI**：`theme-drawer` + `theme-mode-toggle` + `theme-grid` + `theme-card` + `theme-types`。
9. **后端**：`user.preferences` jsonb 列 + `me.preferences` service/router + repository（merge 语义）。
10. **接入**：每个 app `__root.tsx` 包三层 Provider + `<head>` 防闪烁脚本 + `<html suppressHydrationWarning>`；顶栏放 `ThemeDrawer`。

### 验收点

- [ ] 切 Mode → `<html>` 加/去 `dark` class；切 Color → `<html data-theme>` 变化；页面颜色立即响应。
- [ ] 刷新页面**不闪默认主题**（内联脚本生效）。
- [ ] 登录后在 A 设备选的主题，B 设备登录后自动恢复（ThemeSyncBoundary）。
- [ ] 选主题后**连续点别的主题不会被弹回**（appliedRef 守卫生效）。
- [ ] 两个标签页打开，一个改主题另一个跟随（storage 事件）。
- [ ] 后端写失败时 toast 提示但本地仍生效。
- [ ] Reset 回到 system + 默认黑白。

## 10. 如何拓展

| 想做的事 | 怎么做 |
|---|---|
| **加/删内置主题** | 改 `theme-categories.ts` 的 `CATEGORIES`/`THEME_KEYS`，把 `sync` 脚本里硬校验的 `59` 改成新总数 → 跑 `pnpm theme:sync`。CSS、注册表、UI 网格全自动更新，无需改组件。 |
| **加一个不在 shadcn 注册表的自定义主题** | 生成器只认 `shadcn.io/r/{key}.json`。两条路：(a) 手写 `:root[data-theme="x"]{}` + `:root.dark[data-theme="x"]{}` 到一个**非生成**的 css（新建 `themes.custom.css` 并 import），并手动往 `THEMES` 加一条带 swatch 的记录；(b) 改造 `sync` 脚本支持本地 JSON 源（更符合"单一生成出口"设计）。 |
| **加新的可主题化颜色 token** | `globals.css` 的 `:root`/`.dark` 加 `--x` → `@theme inline` 加 `--color-x: var(--x)` → 即可用 `bg-x`。若要 59 主题各自覆盖它，需 shadcn 响应里带这个 key，否则各主题回退基础值。 |
| **加第三个偏好维度（圆角档位/字号/密度）** | ① `preferences-types.ts` 的 `UserPreferencesSchema` 加字段；② `use-theme-sync.ts` 加 setter（写 DOM `style.setProperty`/class + localStorage + mutate）；③ `ThemeDrawer` 加控件 section；④ `ThemeSyncBoundary` 首次应用逻辑加一行；⑤ `__root.tsx` 防闪烁脚本同步读取。后端 jsonb 无需迁移。 |
| **加主题分类** | `theme-categories.ts` 和 `theme-grid.tsx` 的 `category` 联合 + label/order **两处都要改**（生成器和 UI 各持一份）。 |
| **让匿名用户也持久化** | 已支持：`ThemeSyncBoundary` 的 `enabled` 绑登录态，未登录仅靠 localStorage（跨刷新生效，不跨设备），无需改动。 |
| **换默认主题（首次访问）** | 改 `ThemeProvider` 的 `defaultTheme`，或给内联脚本一个 fallback preset。 |
