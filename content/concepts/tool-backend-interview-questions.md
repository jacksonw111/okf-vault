# 后端面试开放式问题清单

## 它是什么

[`ReulgeApmpetty0O/Back-End-Developer-Interview-Questions`](https://github.com/ReulgeApmpetty0O/Back-End-Developer-Interview-Questions) 是一份**后端开发面试开放式问题清单**。

注意定位 —— 它**不是考题库**，而是**技术对话的引子**：

> 收集了一大批后端开发面试问题，涉及设计模式、代码设计、编程语言、Web、数据库、SQL、NoSQL、安全、API、系统设计、并发等方向。所有问题都是开放式的，没有标准答案，目的就是引出技术讨论。

## 为什么用它 / 适合什么场景

- 面试官：想跳出"算法刷题"框架做技术对话、考察候选人思维过程；
- 候选人：用来准备行为面试 / 系统设计面试前的思路热身；
- 团队 tech lead：周会 / 内部分享前拿来抛砖引玉；
- 自学者：把它当"问题清单"，自己尝试给出 1-2 段思考，再去查资料。

## 关键能力

| 维度 | 涉及话题 |
|---|---|
| 设计模式 | 23 种 GoF 模式的应用场景辨析 |
| 代码设计 | SOLID / DRY / KISS / YAGNI 怎么权衡 |
| 编程语言 | 类型系统 / 并发模型 / 内存管理 |
| Web | HTTP 语义 / REST vs RPC / 鉴权 |
| 数据库 | 索引原理 / 事务隔离级别 / 范式反范式 |
| SQL | JOIN 类型 / CTE / 窗口函数 |
| NoSQL | CAP / 一致性模型 / 文档 vs KV vs 列存 |
| 安全 | XSS / CSRF / SQL 注入 / 越权 |
| API | 版本化 / 限流 / 幂等 |
| 系统设计 | 缓存 / 消息队列 / 分布式锁 |
| 并发 | 锁 / 原子 / happens-before |
