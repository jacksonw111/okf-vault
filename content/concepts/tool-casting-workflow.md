# Casting-Workflow（番茄小说短篇生成）

## 它是什么

一个纯本地运行的 Python 小说创作管道，目标是绕过朱雀等查重工具的反查。它的核心思路很「反直觉」：

1. 拿同一类型的 5 篇小说，分别抽取**指纹**（人物、情节、句式、高频词）。
2. 取这些指纹的**交集**——这就是该类型的「**类型公约数**」。
3. 用公约数引导 LLM 生成全新故事。

因为公约数属于「类型本身」而不是任何单一作者，所以查重工具**无法把生成结果追溯到任何一篇源文**。

## 为什么用它

- **本地优先**：不需要联网上传稿件，不依赖任何云端服务。
- **绕查重思路优雅**：不靠「洗稿」「替换近义词」，而是**数学化地脱离单源**。
- 适合在番茄小说这类「查重严格、更新频次高」的平台上做产能工具。

## 关键步骤

| 步骤 | 干什么 |
|------|--------|
| 指纹抽取 | 人物 / 情节 / 句式 / 高频词，多维向量化 |
| 公约数 | 多篇指纹求交集 |
| 引导生成 | 把公约数作为 prompt 上下文喂给本地 LLM |
| 验证 | 生成结果回比查重阈值 |

## 边界

- 解决的是「**类型级**」的查重绕开问题，不处理「**段落级**」抄写检测。
- 生成质量上限仍取决于本地 LLM 能力 + 公约数质量。
